docker镜像过大

  • 在 Dockerfile 中, 每一条指令都会创建一个镜像层, 会增加整体镜像的大小. 合并多个指令串起来执行, Dockerfile 中的 RUN 指令通过 && 和 / 支持将命令串联在一起, 可以省不少镜像层

  • 选用更小的基础镜像, 比如运行 SpringCloud 项目，不必用 jdk , 直接选用 jre 镜像即可, 可以节省50%的内存甚至更多
